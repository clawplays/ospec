# OSpec バージョンアップ説明書

本文書は、このリリースリポジトリのバージョンアップ手順を記録するためのものです。今後、単に「バージョンを上げる」または「1つ上のバージョンに上げる」と言及された場合、明示的に `minor` または `major` を上げることが指定されない限り、デフォルトで **patch +1** として処理します。

## 適用範囲

- OSpec CLI リリースリポジトリ
- リリースバージョン番号と外部表示用バージョン番号を同期させる必要があるシーン
- ユーザー自身が Git コミットを行う（本手順では自動コミットを行わない）

## デフォルトルール

- `バージョンを1つ上げる` = `patch +1`
- 例：`0.6.0 -> 0.6.1`
- ユーザーが明示的にマイナーバージョンを上げると言った場合は `minor` とします
- ユーザーが明示的にメジャーバージョンを上げると言った場合は `major` とします

## 同期が必要なファイル

バージョンアップのたびに、以下の場所を確認して同期する必要があります：

1. `package.json`
2. `package-lock.json`
3. `dist/cli.js`
4. `docs/usage.md`
5. `docs/usage.zh-CN.md`

## 標準操作手順

### 1. 現在のバージョンの確認

まず、以下のファイルで現在の CLI バージョンを確認します：

- `package.json`
- `package-lock.json`
- `dist/cli.js`
- `docs/usage.md`
- `docs/usage.zh-CN.md`

### 2. `package.json` および `package-lock.json` のアップグレード

推奨コマンド：

```bash
npm version patch --no-git-tag-version
```

マイナーバージョンまたはメジャーバージョンを上げる場合は、それぞれ以下を使用します：

```bash
npm version minor --no-git-tag-version
```

```bash
npm version major --no-git-tag-version
```

### 3. CLI 定数バージョンの同期

`dist/cli.js` 内の `CLI_VERSION` を新しいバージョンに変更します。

### 4. 外部ドキュメント内のバージョン表示の同期

以下のファイル内の CLI バージョン表示を更新します：

- `docs/usage.md`
- `docs/usage.zh-CN.md`

メインの README には現在 CLI バージョン番号が表示されていないため、ここでは README の修正は不要です。

CLI バージョンのみを更新し、別途要求がない限り、他のプロトコルやスキルのバージョンは変更しません。

### 5. 最小限の検証の実施

少なくとも以下を実行してください：

```bash
node dist/cli.js --version
```

出力される新しいバージョンが正しいことを確認します。

必要に応じて、追加で以下を実行することも可能です：

```bash
npm run release:smoke
```

### 6. 完了報告

完了後、ユーザーに以下の内容を報告します：

- 今回のバージョンがいくつからいくつに上がったか
- どのファイルを修正したか
- 最小限の検証を行ったか
- Git コミットはユーザー自身で行うこと

## リポジトリの規約

- 今後、ユーザーが単に「バージョンを上げる」または「バージョンを1つ上げる」と言った場合、デフォルトで `patch +1` とします
- ユーザーから明示的な要求がない限り、Git タグの自動作成や Git の自動コミットは行いません
- 将来リポジトリに新しいバージョン表示場所が追加された場合は、本文書にも追記してください
